# 校园RPG冒险游戏 — 任务分工与贡献度

## 项目概述

基于 **C++ Qt Widgets** 开发的校园题材轮回 Roguelike 回合制 RPG 游戏。玩家创建角色后经历排课、寒假、天使商店、七层地窟探索，团灭后进入轮回保留成长属性，最终击败伪典校长·零通关。

## 一、实现技术与功能

### 开发环境与技术栈

| 技术 | 说明 |
|------|------|
| 语言 | C++17 |
| GUI框架 | Qt 5.15.2 (Widgets) |
| 构建工具 | qmake + MinGW 8.1.0 |
| 版本控制 | Git + GitHub |
| 资源制作 | AI绘画工具生成全部场景/角色/敌人素材 |
| 数据存储 | INI文本存档（4个槽位，可读可调试） |

### 面向对象设计

| 特性 | 实现 | 说明 |
|------|------|------|
| **封装** | `CharacterData`/`EnemyData`/`ItemData`/`TaskData` | 游戏核心数据均封装为结构体或类，属性通过公共接口访问 |
| **继承** | `Profession → StudentProfession/IceMageProfession/PaladinProfession/BlesserProfession/BloodWarriorProfession/MagicianProfession` | 6个职业派生自抽象基类，各自实现 `skills()` 和 `baseStats()` |
| **多态** | 虚函数 `Profession::skills()` / `Profession::baseStats()` | 通过基类指针动态调用不同职业的技能和属性 |
| **虚析构** | `virtual ~Profession() = default` | 确保通过基类指针安全释放派生类对象 |

### STL 容器使用

| 容器 | 用途 | 位置 |
|------|------|------|
| `std::vector<std::unique_ptr<Profession>>` | 保存6个职业多态对象，管理生命周期 | MainWindow.h |
| `QVector<CharacterData>` | 队伍角色列表，支持编队排序 | MainWindow.h |
| `QVector<ItemData>` | 背包物品列表（上限40格） | MainWindow.h |
| `QVector<TaskData>` | 任务列表，支持状态追踪 | MainWindow.h |
| `QVector<RoomData>` | 地窟房间列表（每层5-7房间） | MainWindow.h |
| `QVector<EnemyData>` | 战斗敌人列表（支持A/B变体） | MainWindow.h |
| `QMap<QString, EnemyData>` | 图鉴：按名称快速查找已遭遇敌人 | MainWindow.h |
| `QMap<QString, QString>` | 角色装备槽：槽位→装备名 | CharacterData |

**评定依据：**

1. **谢文浩 30%** — 承担组长职责，负责最核心的战斗引擎（约600行复杂逻辑）、地窟生成算法、职业继承体系设计。战斗系统是整个游戏的技术难点，涉及6种异常状态、30+种敌人AI、120种技能效果的实现。同时负责Git仓库管理和代码合并。

2. **黄金弘 24%** — 负责商店（天使+恶魔双系统）、任务系统和数据持久化三大模块。存档序列化涉及8个序列化函数（角色/物品/任务/图鉴的读写），是整个项目数据完整性的核心保障。

3. **张轩 23%** — 负责角色管理、背包、装备和编队四大模块。装备系统涉及4种槽位的穿戴逻辑和属性加成计算；编队系统实现了灵活的前后排配置和排序功能。

4. **张申桥 23%** — 负责全部Qt图形界面（7个Tab页布局+样式）、排课系统UI（14格课表+实时预览）、端到端测试、项目文档和资源管理。虽然代码行数占比略低，但UI开发的工作量和72张素材的整理同等重要。
